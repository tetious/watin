I’ve been meaning to write a roadmap for WatiN for a while now, finally sat down to do this. The releases are both feature and time driven. Every new release needs at least one or two major new features before it will be released. The release cycle will be 2 month. This gives me:

-          2 weeks to sit back and enjoy the download count of the previous release J.

-          4 weeks to do the programming.

-          and 2 weeks to make it release ready.

 

And remember, this roadmap states my intentions but it’s in no way meant to rule out any changes in planned features, release dates or what so ever. So if you have something that’s not on the list (I can think of many other things) please let me know through the feature request tracker on SourceForge.

 

WatiN 0.9.0 (released on 31 October 2006)

Read the release notes here.

 

WatiN 1.0.0 (scheduled for the end of December 2006)

-          Support for the IE 7 browser. (Note that WatiN 0.9 works fine with IE 7). DONE

-          Support more standard IE dialogs:

1.      certificate dialog

2.      download dialog

3.      print dialogs DONE

4.      security alert dialog

-          Support for ie.Element(findBy).Exists DONE

-          Support for ie.Element(findBy).WaitUntilExists DONE

-          Improve support for JAVA injected elements by calling WaitUntilExists internally. This is a common scenarios when AJAX technology is used. DONE

-          Support filtering of Element collections. DONE

-          Improved SelectList support.

-          Introduce a parameterization class to set things like flashing, time out length etc…

-          Screenshot support.

-          An installer created with WIX (and SharpDevelop WIX support in version 2.1).

-          Documentation complete. IN PROGRESS

-          More unit tests. IN PROGRESS

 

Might be in this release:

-          A Mock SMTP server for sending email to from an application (this requires you can set the SMTP server your app under test send the mail to). You can use this Mock SMTP server in your unit, functional or acceptance tests to verify mail is send and to check the To, Cc, Bcc, Subject, Body and Attachment(s). We are currently developing this and thought it might be useful to distribute with WatiN. IN PROGRESS

-          An NUnit 2.4 extension: “STAFixture” to remove the need for the additional .config file to set the ApartmentState to STA.

-          A script recorder. I have changed WatirMaker (from Scott Hanselman) so it now outputs NUnit/WatiN code. It works but with many restrictions (if your interested in a copy, let me know). Maybe it’s better to wait for the release of WatirRecorder (from Scott and Rutger Smit) on www.openqa.org before releasing a recorder with WatiN.
UPDATE: Richard Griffin has released a prototype of WatiNRecorder++ here.

 

WatiNFixture:

-          Jeff Parker has launched WatinFixture as a separate project on SourceForge. You can find it here.

 

After the 1.0.0 release

-          Vista compatible (Thanks to Bruce McLeod for his initial testing of WatiN on Vista RC1, see his weblog for more info). IN PROGRESS

-          Create a .Net 3.0 distribution
