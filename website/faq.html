Following are some frequently asked questions about WatiN

 

Are you going to port WatiN to the .Net 2.0 framework?

Yes. WatiN version 0.8.0 will have a distribution for the .Net 1.1 and .Net 2.0 frameworks. In the meantime you could port WatiN to .net 2.0 yourself. This is already successfully done by some other WatiN users.

 

Which versions of Internet Explorer are supported?

Currently only Internet Explorer 6 but it will run without any problems with Internet Explorer 7 beta 3.

 

Are you planning to support FireFox?

No, not within the near future. The coupling between WatiN and Internet Explorer is (currently) too tied to make this work.

 

Will there be a recorder for WatiN scripts?

Probably. Scott Hanselman and Rutger Smit have created an (open source) recorder, called WatirRecorder++ on www.codeplex.com (but soon on www.openqa.org) . Maybe this project could be extended or used as a basis for creating a WatiN recorder. This will be investigated in Q4 of 2006.

 

I get an “invalid cast exception” when creating an IE instance with WatiN 0.7. What’s wrong?

Make sure the thread your code is running is a single threaded apartment (STA) thread. For more information on this subject read this article. WatiN 0.8.0 (and later) will throw a ThreadStateException to help you remind to set the apartmentstate to STA.

 

Can I help in anyway?

Yes! WatiN is an open source project so your contribution is highly appreciated. Please subscribe to the development mailing list to participate in the development of WatiN.

 

Under which open source license is this project distributed?

The LGPL license. See http://www.opensource.org/licenses/lgpl-license.php  for more information about this license.

 

What does WatiN mean and how do you pronounce it?

WatiN (pronounced as “what-in”) stands for “Web Application Testing In .Net”.

  • In Poland it means: Garage.
  • In Finland (and maybe some other countries) it means: Watt.
