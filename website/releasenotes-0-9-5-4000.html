This page contains the release notes for WatiN 0.9.5.4000, released on 31 December 2006.

 

Changes in general

·         This release fully supports Internet Explorer 7 and has been tested with the international and Dutch versions of IE on both Windows XP (SP2) and Windows 2003 Server. Note that there are still issues running WatiN (with IE7) on Vista RTM.

·         Added support for automating the print dialog. (Thanks to Marcel Veselka for the initial implementation).

·         Added support to (better) handle injection or removal of DOM elements by client side JAVA code. Since AJAX technology uses this extensively and the number of websites that uses AJAX is growing rapidly, most of the development time for this release has been invested in adding the following features.

 

To check if an element exists on a webpage you now can make use of:

ie.Button(findBy).Exists;

 

To wait until an element exists on a webpage you can use:

ie.Button(findBy).WaitUntilExists(); // timeout is set to 30 seconds

ie.Button(findBy).WaitUntilExists(timeout);

 

Note that WatiN also calls WaitUntilExists internally when looking up an element on a webpage. So you only need to call WaitUntilExists(timeout) if the default timeout period of 30 seconds isn’t long enough.

 

Please read the following if you already use WatiN.

 

To support these new functionalities, WatiN’s element lookup mechanism has been changed from “early binding” to “late binding”. This means WatiN delays the element look up until it’s actually needed (when a property or method of the element is invoked). Once the binding is successful, the reference to the element on the webpage will be reused for subsequent property and methods calls. This has resulted in a slightly different behaviour which can best be explained with some examples. Take the following line of code:

 

Button button = ie.Button(findBy);

 

WatiN 0.9.0 and earlier: Element lookup was done immediately (An ElementNotFoundException was thrown if the element wasn’t found).

WatiN 0.9.5 and higher: No element lookup is done yet (Element look up is delayed until a property or method of the element is invoked).

 

Behaviour change in the following line of code:

 

Button button = ie.Button(findBy).Click;

 

WatiN 0.9.0 and earlier: Element lookup was done immediately and the button was clicked. An ElementNotFoundException was thrown immediately if the element wasn’t found.

WatiN 0.9.5 and higher: Element lookup is done immediately and the button is clicked. If the element isn’t found within 30 seconds, an ElementNotFoundException is thrown.

 

Yet another example.

public bool ButtonExists(Document document, Attribute findBy)

{

  try

  {

    Button button = document.Button(findBy);
    return true;

  } 

  catch(ElementNotFoundException)

  {

    return false;

  }

}

 

This code was previously needed to check if an element existed on a webpage. WatiN 0.9 and earlier would throw an ElementNotFoundException if the element could not be found. But with WatiN 0.9.5 and later the previous code will ALWAYS return true since element look up is delayed until a property or method of the element is invoked. To make your code work again you need to change your code to:

 

public bool ButtonExists(Document document, Attribute findBy)

{

  return document.Button(findBy).Exists;

}

 

To avoid problems due to this behavioural change; please check your code and change it if you use any constructs as described in the previous examples.

 

·         Added support for filtering a collection of elements.

 

ButtonCollection buttons = ie.Buttons.Filter(findBy);

 

·         Added support to check if an element exists in a collection.

 

bool exists = ie.Buttons.Exists(findBy);

 

·         Added support to check if an instance of Internet Explorer already exists.

 

bool exists = IE.Exists(findBy);

 

·         Elements can now be found by none string attributes, like sourceIndex

 

Button button = ie.Button(Find.ByCustom(“sourceindex”, “25”));

 

·         Added support for regex as an overload on all elements methods. Now you can write this, somewhat shorter, code when using a regular expression to find an (button) element by it’s Id:

 

ie.Button(new Regex(@"[a-zA-Z0-9\$]*SearchBarTextbox").Click

 

·         Support has been added to access all type of image elements (<image /> elements as well as <input type=image /> elements) on a webpage through

 

Image image = ie.Image(findBy);

 

and

 

ImageCollection images = ie.Images;

 

Accessing images of type input through ie.Button is also still supported.

 

WARNING: If you use both element types on the same webpage and you currently validate the number of images through ie.Images.Length, the result will be differ from the results with previous WatiN versions.

 

·         An Element can be casted to a Button Type. In the next release this will be supported for all elements

 

Element element = ie.Element(findBy);

Button button = new Button(element);

 

·         When calling ie.WaitForComplete, WatiN now also waits until all nested frames and iframes are loaded.

·         When calling FireEvent on an element in a HTMLDialog or when FireEvent is called internally by WatiN, the mousebutton property of the event object is set to left button.

·         The return values of Element.Parent, Element.NextSibling and Element.PreviousSibling can be casted to type ElementsContainer. This only works well if you know for sure the returned Element contains other elements which is true for table, tablerow, tablecell, para, div etc.

·         None string attributes, like sourceIndex, can now be accessed through Element.GetAttributeValue("sourceIndex")

·         Table.FindRow now ignores rows which contain anything else then <TD /> elements.

 

 

API Changes

 

Namespace WatiN.Core:

 

All element collection classes like ButtonCollection, TextFieldCollection etc..:

·         Added Filter(findBy) method. Returns a typed collection with elements that match the given findBy.

·         Added Exists(findBy) method. Returns true if an element in the collection matches the given findBy.

 

BaseElementCollection class (and subclasses)

·         Introduced to reduce the code necessary to create a typed collection.

 

Element class (and subclasses)

·         Added Exists property. Returns true if an element exists.

·         Added WaitUntilExists method. This wait 30 seconds before it throws a TimeoutException.

·         Added WaitUntilExists(int) method. This wait the given seconds before it throws a TimeoutException.

·         Added HTMLElement property. Returns the “raw” DOM element that is wrapped by WatiN.

 

ElementFinder class

·         Introduced to make the “late binding” of an element possible.

 

ElementTag class

·         Renamed, was TagType.

·         Introduced to define which element tags map to a WatiN element.

 

SimpleTimer class

·         Introduced to make it easier to handle time out logic.

 

Namespace WatiN.Core.Interfaces:

 

IElementsContainer

·         Added overloads for every supported element to accept a RegEx instance as an argument.
