Releasenotes WatiN 2.0 beta 1

This page contains the release notes for WatiN 2.0 beta 1, released on 30 March 2009.

You can download this release here.


Changes in general

This first beta release of WatiN 2.0 is a major step in multi browser support and offers many new features as well. All WatiN unit tests previously only running tests with IE are now browser agnostic and so verify and enforce the same functional behavior across all browser implementations. Currently the following browsers are supported:

IE6, IE7, IE8 (IE8 mode has some issues), Firefox 2.x and 3.x, Chrome (experimental and a work in progress).

I would like to thank my fellow committers for their work!
Jeff Brown has completely revamped WatiN's internals and added some exciting new features.
Edward Wild for his initial work on Firefox and adding Chrome support, which is still a work in progress).
And of course thanks go out to all of you who reported a bug or did send in patches!

Should I upgrade? My personal opinion: YES (but I do have a slight bias). Even if you don't intend to test on multiple browsers this release will offer you many new features. This release is stable and is in use for a while on severall test projects. But before you replace your current WatiN version, please read the Breaking Changes and Known Issues sections to make an informed decision your self.

What's next? We will continue working on WatiN 2.0 to fix the known issues and incorporate community feedback. We intend to do a release at the end of every month, working towards a final release in june or july. After the 2.0 version has become final, we will set out a new road map for a next release.


Functional changes

Jeff Brown has added support for a Control and Page model. I will blog about this new feature with some examples soon. Mean while read the excellent documentation in the help file.

Jeff Brown also added the possibility to add Elements to WatiN. For more details and an example see this blog post.

Added support for running WatiN tests from a service, a locked workstation or a remote desktop with no UI. In previous releases this resulted in hanging tests due to sending text to one of the following dialog handlers: FileUploadDialogHandler, FileDownloadHandler, PromptDialogHandler and LogonDialogHandler.

Added support for Internet Explorer 8. See also known issues section about issue with IE8 mode.

Finally Ie.AttachtToIE(constraint) and new IE(inNewProcess) do work on Vista and Windows 2008.

Added DialogHandlerHelper. This dialoghandler can be used when it is unclear which DialogHandler should be used to handle a dialog. The property CandidateDailogHandlers will contain a list of dialoghandlers which could handle this dialog. Following and example on how to use the DialogHandlerHelper: var helper = new DialogHandlerHelper();
using (new UseDialogOnce(ie.DialogWatcher, helper)
{
  ie.Button("showAlert").Click();
}
foreach(string dialogHandler in helper.CandidateDialogHandlers)
{
  Console.WriteLine(dialogHandler);
}

Implemented Element.Equals(element) without needing to set the Id attribute of an html element. This is especially handy when doing something like: var isRowOfTable = tableRow.ContainingTable.Equals(table);

Link has become an ElementContainer so you can get to any contained element inside the link (like for instance an Image).

Added new options to the Find class: Find.Any, Find.None, Find.ByExistanceOfRelatedElement and Find.ByTextInColumn

All collections inherit from BaseComponentCollection, which adds Filter(constraint), First(constraint) and Exists(constraint) support to IECollection, FrameCollection and HTMLDialogCollection.

Added Element.SetAttributeValue(attribName, value). Also added a setter property to Element.Id.

PrintDialogHandler will now recognize print dialog on Vista and Windows 2008.

Improved reporting of comparer conditions.

Added Logger.LogDebug() to make it possible to distinguish between actions (= functional) and debug (= technical) messages.

Added a Peek() method to the AlertAndConfirmDialogHandler so you can see the first item without Popping it. Thanks to Chris Donnelly for this patch!

When calling element.GetAttributeValue(eventName) the text value of the event attribute will be returned. Thanks to Daaron Dwyer for this patch!


Technical changes

This release only supports .Net 20 and up.

Greatly simplified multi browser support when compared with the CTP releases.

WatiN.Core assembly is now strong-signed (and so are Interop.SHDocVw.dll and WatiN.UnitTests.dll).

Fixed the issue where viewing a nonexistent Element in a watch was causing the debugger to time out and refuse to let you view anything else (i.e. the "Function evaluation timed out." error). Added a DebuggerDisplayAttribute that shows the constraint and whether Exists is true or false. Thanks to Chris Donnelly for this patch!

Also read the following breaking changes section cause these are mainly technical changes.


Known issues

  • When running your tests in Internet Explorer 8 and your page(s) enforce IE8 mode, there will be problems with firing events from WatiN on elements in these pages.
  • The IE.ReOpen functionality is currently broken.
  • Chrome support is still very experimental.
  • DialogHandler support for Firefox is (still) missing.
  • HtmlDialog support for Firefox 3.x is missing.
  • Find.Near and ProximityTextConstraint not supported by FireFox and Chrome.
  • FileUpload.FileName returns on file name without path on Windows 2008 and with "fakepath" on IE8.


Breaking changes

This list should gives you a good indication of what has changed but doesn't mention every detail.

  • On the Table, TableBody, TableRow and TableCell classes the properties and methods *DirectChild* are renamed to Own*. Also the Parent* properties have been renamed to Containing*.
  • IE.Settings has been replaced by a new static Settings class.
  • The internals of WatiN have been redesigned by Jeff Brown, many of these classes (and interfaces) to support finding an element are revamped and renamed. But this probably won't affect your test code.
  • The introduction of the Comparer{T} class made ICompare and ICompareElement obsolete.
  • ProximityTextConstraint is no longer an AttributeConstraint.
  • PredicateElementComparer and PredicateStringComparer are folded together into PredicateComparer.
  • IElementsContainer is renamed to IElementContainer.
  • INativeElement.Object and INativeDocument.Object are gone. To get underlying Html DOM object, cast to IEElement / IEDocument or FireFox equivalent then use specialized methods for each type.
  • Table.TableRows now returns all rows including in nested tables. Use Table.OwnTableRows instead. This change was made for consistency across all IElementContainer implementations.
  • Renamed Element(string tagName,....) to ElementWithTag(string tagName,....).
  • Button no longer supports elements with tagname input of type image. Use Image instead.
  • Removed Element.HTMLElement property, use Element.NativeElement and cast it to a specific implementation. var htmlElement = ((IEElement)element.NativeElement).AsHtmlElement;
  • Document.FireEvent(DispHTMLBaseElement, eventName) is removed.


Breaking changes: CTP releases to beta 1

This list should gives you a good indication of what has changed but doesn't mention every detail.

  • Each and every Element implemented a specific interface (for example ITextField). Since these interfaces are no longer needed for multi browser support, these interfaces are dropped. Instead all properties and methods on the elements are made virtual so they can be overridden if you need to.
  • Support for Enum BrowserType is dropped. Since WatiN can support different browsers and you can add your own browser implementation as well, having a fixed Enum with a fixed list of supported browsers would lock things down again.
  • Support for BrowserFactory class is dropped. Same reason.
